(no version information, might be only in CVS)

xcc_new_module_invoke -- Creates a new resource for invoking a module

Description

resource xcc_new_module_invoke ( string uri [, resource link] )

Sets up a module to be invoked. The uri should be set to the uri where the module can be found in the database. External variables can be set on the returned request variable.

Parameters

uri

The uri should be set to the uri where the module can be found in the database or on the filesystem.

link

The connection to the Mark Logic server. If the link identifier is not specified, the last link opened by xcc_connect() is assumed. If no such link is found, an E_WARNING level warning is generated.

Return Values

Returns a request resource that can be passed into xcc_submit_request()()

Examples

Example 1. xcc_new_module_invoke() example

<?php
xcc_new_session($user, $password, $host, $database, $port)
    or die("Counld not connect to server: " . xcc_error());

$stmt = xcc_new_module_invoke('http://example.com/sample.xqy')
    or die("Can't invoke module: " . xcc_error());

$results = xcc_submit_query($stmt) or die("Can't execute query: " . xcc_error());
while($item = xcc_fetch_next($results)) {
    echo $item['data'], "<br/>";
}
?>
