xcc_add_permission -- Add a permission to a document being inserted

Description

bool xcc_add_permission ( resource document, string role, long permission )

When a document is being inserted into the database, this function can be used to grant the specified role a permission on that document.

Parameters

document

The resource returned from xcc_content_from_file() or xcc_content_from_string().

role

The name of the role to grant the permission to. These roles are configured in the server themselves and the insert will error if the role does not exist.

permisssion

The permission that will be given to the specified role for this document.

Must be one of: XCC_PERMISSION_READ, XCC_PERMISSION_INSERT, XCC_PERMISSION_UPDATE or XCC_PERMISSION_EXECUTE

Return Values

Returns TRUE on success or FALSE on failure.

Examples

Example 1. xcc_add_permission() example

<?php
$doc = xcc_content_from_file($file, $uri) or die("Can't connect: " . mysql_error());

xcc_add_permission($doc, "read-role", XCC_PERMISSION_READ);

// inserts the document, returns the number of bytes inserted
$bytes = xcc_insert_content($doc) or die("Failed to insert document: " .  xcc_error());

echo "Inserted $bytes bytes from file<br/>";

xcc_close();
?>
