xcc_content_from_file

(no version information, might be only in CVS)

xcc_content_from_file -- Inserts a new document into the database from a file

Description

resource xcc_content_from_file ( string file, string uri [, resource link] )

Creates a new resource for inserting a document into the database from a file on the local filesystem. This function behaves exactly like xcc_content_from_string() except it reads from a file instead of a string.

Parameters

file

The path to the local file on the filesystem.

uri

The uri that the document will be given once inserted into the database.

link

The connection to the Mark Logic server. If the link identifier is not specified, the last link opened by xcc_connect() is assumed. If no such link is found, an E_WARNING level warning is generated.

Return Values

Returns a statement handle that can be passed into xcc_insert_content()().

Examples

Example 1. xcc_content_from_file() example

<?php
xcc_new_session($user, $password, $host, $database, $port) or die("Counld not connect to server: " . xcc_error());

// create a new document insert handle for the file
$doc = xcc_content_from_file($file, $uri) or die("Failed to created insert handle: " . xcc_error());

// inserts the document, returns the number of bytes inserted
$bytes = xcc_insert_content($doc) or die("Failed to insert document: " . xcc_error());

echo "Inserted $bytes bytes from file<br/>";

xcc_close();
?>
